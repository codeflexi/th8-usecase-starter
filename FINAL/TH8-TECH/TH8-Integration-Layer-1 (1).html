Framework 1

TH8 Agentic Integration Stack™

เลเยอร์กลางที่ทำให้ ERP/WMS/CRM/LINE/IoT คุยกับ Agent ได้อย่างปลอดภัย ใช้แนวคิด “Semantic + Rules + Workflow” แทนการยิง API ตรง ๆ ด้วย Prompt

สิ่งที่องค์กรจะได้

• ใช้ระบบเดิมต่อได้ ไม่ต้องรื้อ

• ต่อ Agent เพิ่มทีหลังได้ โดยไม่ยุ่ง Code ระบบหลัก

Conceptual View

ชั้นล่าง: System Connectors

Connector สำหรับ ERP, WMS, CRM, Accounting, LINE OA, IoT พร้อม Mapping Table, Validation Rule, Error Handling

ชั้นกลาง: Business Object & Policy

แปลงข้อมูลดิบเป็น Business Object เช่น Order, Invoice, Trip, Ticket ผูกกับ Policy/Limit/Approval ที่องค์กรใช้อยู่จริง

ชั้นบน: Agent API

ให้ Agent เรียกใช้ในรูปแบบ “task” เช่น create_sales_order(), get_stock_status()

ตัวอย่าง Use case

ตัวอย่าง: Sync Order จาก LINE OA → ERP

Agent ไม่ยิง API ตรง แต่เรียกผ่าน Integration Stack ที่คอยเช็ก Master, Credit, Policy ก่อนเสมอ

  • • ลดโอกาสยิงข้อมูลผิด เช่น ลูกค้าไม่มีเครดิต, ส่วนลดเกินสิทธิ์
  • • ถ้าไม่ผ่าน Rule → โยนเข้า Exception Library และแจ้งคน
  • • ต่อ Agent เพิ่มในอนาคตได้ โดยใช้ Stack เดิม

Framework 2

TH8 3-Layer Agent Architecture™

โครงสร้างการออกแบบ Agent ที่ไม่ให้ทุกอย่างไปรวมกันใน “ตัวเดียวที่ทำทุกอย่าง” แต่แยกเป็น 3 ชั้น: Strategy / Task / Integration เพื่อให้ดูแลง่าย ขยายได้ และควบคุมความเสี่ยงได้

Layer 1 · Strategy / Orchestration

กำหนด “เป้าหมาย + เส้นทางการทำงาน”

เช่น “ลด Lead time จาก Order → Delivery” หรือ “ปิดงบ FP&A ให้เร็วขึ้น 30%” เลเยอร์นี้คือที่อยู่ของ Business Flow / State Machine

Layer 2 · Task / Specialist Agents

Agent ย่อยตามหน้าที่

เช่น FP&A Copilot, Inventory Agent, Route Planner, Quality Agent แต่ละตัวมีขอบเขตชัดเจน, Metric ชัด และมี Human-in-loop ที่เหมาะสม

Layer 3 · Integration & Guardrail

เชื่อมกับระบบและคุมความเสี่ยง

ทุก Action ที่แตะ ERP/WMS/Accounting ต้องผ่าน Rule/Limit/Approval และมี Audit log ดูย้อนหลังได้

Framework 3

TH8 Knowledge Copilot™

แปลง Manual, Policy, SOP, Contract, FAQ, Case log ให้กลายเป็น “Copilot ภาษาไทย” ที่ตอบได้ทั้งฝั่งหน้างานและ Back-office – พร้อมรู้ว่าควรอ้างอิงเอกสารไหน

Staff View · ถาม–ตอบได้เหมือนถามรุ่นพี่

ตัวอย่างคำถาม

“ลูกค้ากลุ่ม Dealer A ขอส่วนลดเพิ่ม ถ้าสั่งเกิน 500,000 บาท ผมให้ได้สูงสุดเท่าไหร่ถ้าจ่ายเครดิต 60 วัน?”

คำตอบจาก Knowledge Copilot

• กลุ่ม Dealer A: ส่วนลดมาตรฐาน 10–12% • ถ้ายอดเกิน 500,000 บาท และเครดิต 60 วัน → ส่วนลดสูงสุด 14% • ถ้าเกิน 14% ต้องส่งเรื่องให้ผู้จัดการเขตอนุมัติ

Admin View · Document & Exception Library

Document Inventory (ตัวอย่าง)

  • • Policy ส่วนลดตามประเภทลูกค้า (10 เอกสาร)
  • • SOP การอนุมัติส่วนลด และ Credit limit
  • • Template ใบเสนอราคา / สัญญามาตรฐาน
  • • Case ที่เคยให้ส่วนลดพิเศษ + เหตุผล

Exception Library

เก็บกรณี “ผิดจากกติกา” ไว้อัปเดต Policy และใช้สอน Agent ให้เก่งขึ้น

  • • เคสให้ส่วนลดเกิน Policy เพราะต้องเคลียร์ Stock
  • • เคสอนุมัติเครดิตพิเศษให้ลูกค้ากลุ่ม Strategic

Framework 4

TH8 Workflow Sync Engine™

กลไกกลางที่ทำให้ “งาน 1 เรื่อง” วิ่งครบทุกระบบ โดยไม่หล่นกลางทาง เหมาะกับ Flow เช่น Order-to-Cash, Procure-to-Pay, Lead-to-Contract, Incident-to-Resolution

ตัวอย่าง Flow: Order-to-Cash

  1. 1) รับ Order จาก LINE OA / Web / Sales app
  2. 2) Workflow Engine สร้าง Sales Order ใน ERP ผ่าน Integration Stack
  3. 3) เมื่อ WMS ปิดการจัดของ → สร้าง Delivery / DO อัตโนมัติ
  4. 4) เมื่อส่งของแล้ว → สร้าง Draft Invoice และส่งให้ทีมบัญชีตรวจ
  5. 5) เมื่อเก็บเงินครบ → อัปเดตสถานะ AR + Dashboard ฝ่ายบริหาร

สิ่งที่ Engine นี้ Track ให้

Visibility

เห็นว่า Order ไหนติดค้างที่ขั้นตอนไหน เช่น รอจัดของ / รอออกบิล / รอเก็บเงิน

Error & Exception

เคสข้อมูลไม่ครบ, ยิงเข้าระบบไม่สำเร็จ, หรือเครดิตไม่พอ → ถูก log และแจ้งคน
