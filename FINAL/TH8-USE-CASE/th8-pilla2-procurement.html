1) วางแผนซื้อ & ขอซื้อวัตถุดิบ (PR)

Planner / ทีมคลัง

ปัจจุบันมักใช้ Excel / Google Sheet ดูยอดใช้ย้อนหลัง–ยอดขาย–Stock คงเหลือ แล้วคำนวณคร่าว ๆ ว่าควรซื้อเท่าไหร่ ก่อนส่งเอกสารขอซื้อ (PR) เข้า ERP.

  • ดู Forecast / ยอดขายย้อนหลัง
  • เปิด Stock Card ดูคงเหลือในคลัง
  • เช็ค Minimum / Safety Stock
  • กรอก PR ในระบบ ERP หรือแบบฟอร์มภายใน

2) ออกใบสั่งซื้อ (PO) ใน ERP

ERP · Purchasing

ทีมจัดซื้อแปลง PR เป็น PO เลือก Supplier, ราคา, เงื่อนไขเครดิต และกำหนดวันส่งของ.

PO No. Vendor Delivery Date Amount Status
PO-2025-00487 บจก. เหล็กไทยซัพพลาย 05/03/2025 1,250,000.00 Open

3) รับของเข้าคลัง (GR) & จัดเก็บเข้าที่ (Putaway)

เมื่อ Supplier ส่งของมาที่คลัง ทีมรับของจะใช้เอกสารดังนี้:

  • ใบส่งของ / ใบกำกับภาษี จาก Supplier
  • ใบสั่งซื้อ (PO) ปริ๊นจาก ERP
  • แบบฟอร์มรับของ / GR Note

จากนั้นบันทึก GR ใน ERP หรือ WMS และสั่งทีมคลังจัดเก็บเข้าตำแหน่งที่เหมาะสม.

4) รวมชุดเอกสารส่งบัญชี (PO + GR + Invoice)

หลังรับของเสร็จ เอกสารกายภาพจะถูกเย็บชุดส่งให้ฝ่ายบัญชีเพื่อใช้ตั้งหนี้:

  • สำเนา PO จาก ERP
  • ใบรับของ / GR Note ที่เซ็นรับแล้ว
  • ใบกำกับภาษี / ใบอินวอยซ์ จาก Supplier

5) บัญชีตรวจเอกสาร → ตั้งหนี้ (AP) → จ่ายเงิน

Accounting

บัญชีใช้เวลาเช็คตัวเลขและรายละเอียด 3-way match ระหว่าง PO, GR, Invoice ก่อนบันทึกเจ้าหนี้และทำ Payment run / โอนเงินผ่านธนาคาร.

  • เช็คยอดเงิน, ส่วนลด, VAT, เลขที่ใบกำกับภาษี
  • เช็คจำนวนที่รับจริง เทียบเอกสาร–ระบบ
  • กำหนด Due date ตาม Credit term

A) Purchase Planning Agent · แนะนำปริมาณ & Timing ที่ควรซื้อ

AI Agent

Agent ดึงข้อมูลจากยอดใช้จริง, Forecast, Stock ปัจจุบัน และ Minimum level เพื่อเสนอ PR ให้ Planner อนุมัติง่าย ๆ.

  • คำนวณ reorder point และ reorder qty อัตโนมัติ
  • เสนอ Split order ถ้าซัพพลายเออร์ต่างกัน / MOQ ไม่เท่ากัน
  • เตือนล่วงหน้าเมื่อ stock เสี่ยงขาดใน 7–14 วัน

ตัวอย่าง Output:

  "reason": "ยอดใช้เฉลี่ย 130kg/วัน · stock เหลือ 450kg · มี Promotion ต้นเดือนหน้า"
}

B) Receiving Copilot · ช่วยทีมรับของเช็คความถูกต้อง

AI Agent

หน้างานสามารถถ่ายรูปใบส่งของ / ใบกำกับภาษี แล้ว Agent ใช้ OCR ดึงข้อมูลมาเทียบกับ PO ใน ERP ลดเวลาคีย์มือและลด miss-match.

  • ตรวจเลข PO, ชื่อ Supplier, วันที่, เลขที่ Invoice
  • เทียบจำนวน & ราคา กับข้อมูลในระบบ
  • แจ้งเตือนเคส “เกิน/ขาด”, ราคาไม่ตรง, หรือ VAT คำนวณผิด

ตัวอย่าง Alert ถึงหัวหน้าคลัง:

⚠️ PO-2025-00487 · Supplier: เหล็กไทยซัพพลาย
ใบส่งของระบุปริมาณ 3,200 kg แต่ PO/ERP 3,000 kg
กรุณาตรวจสอบก่อนบันทึก GR หรือออกเอกสารรับของเพิ่ม.

C) Slotting Agent · แนะนำตำแหน่งจัดเก็บที่เหมาะสม

Agent ใช้ข้อมูล Movement, น้ำหนัก, ขนาด และความถี่การหยิบ เพื่อแนะนำตำแหน่งเก็บที่ดีที่สุดในคลัง ลดระยะทางเดินและปัญหาคลังรก.

  • จัดกลุ่มสินค้าที่มักถูกหยิบพร้อมกัน ให้เก็บใกล้กัน
  • ให้สินค้าหมุนเร็วอยู่ใกล้จุดโหลด / ทางเดินหลัก
  • เตือนเมื่อ Location ใดเริ่มเกินความจุที่กำหนด

D) AP & Document Reconciliation Agent · ช่วยบัญชี 3-way match

AI Agent

แทนที่บัญชีจะต้องไล่เช็คเอกสารทีละใบ Agent จะดึงข้อมูล PO + GR + Invoice ขึ้นมาเทียบและสรุปว่า “ผ่าน / มีประเด็นอะไร” ให้เลย.

  • OCR ใบกำกับภาษี, ดึงเลขที่, วันที่, ยอดก่อนภาษี, VAT, ยอดสุทธิ
  • เทียบกับ PO และ GR ว่าจำนวน–ราคา ตรงหรือไม่
  • สร้าง AP Entry เบื้องต้นให้บัญชีแค่กดตรวจและอนุมัติ

ตัวอย่างสรุปที่ Agent แสดงในหน้าบัญชี: 

PO-2025-00487 · Vendor: เหล็กไทยซัพพลาย
- จำนวนตรงกัน (PO 3,000 kg · GR 3,000 kg · INV 3,000 kg)
- ราคา/หน่วยตรงกัน 41.67 บาท/กก.
- VAT 7% คำนวณถูกต้อง
✅ พร้อมสร้าง AP 1,250,000.00 บาท · Credit term 30 วัน · Due 04/04/2025

E) Payment & Cashflow Agent · ช่วยจัดลำดับการจ่ายเงิน

เมื่อเจ้าหนี้ถูกบันทึกแล้ว Agent สามารถจัดลำดับความสำคัญของ Payment run ตาม Credit term, ส่วนลดจ่ายก่อนกำหนด และสถานะ Cashflow ของบริษัท.

  • เสนอรายการที่ควรจ่าย “วันนี้ / สัปดาห์นี้” เพื่อไม่ให้หลุด Due
  • ไฮไลต์บิลที่ได้ส่วนลดหากจ่ายก่อนกำหนด
  • เตือนถ้า Payment plan รวมแล้วเกิน Cash-in ที่คาดในช่วงนั้น
