3) ตัด Stock & อัปเดต Delivery ใน ERP

เมื่อของถูกหยิบและโหลดขึ้นลานแล้ว ระบบ WMS/ERP จะสร้างเอกสารตัดสต็อก (Delivery / Issue) และสามารถสร้าง Invoice ต่อได้.

  • ใช้ข้อมูลจริงจากการสแกนบาร์โค้ด / QR ในลานโหลด
  • อัปเดตสถานะ Delivery, Remaining Qty ใน ERP
  • ข้อมูลเหล่านี้กลายเป็นฐานข้อมูลสำหรับ Analytics ต่อไป

4) Material Planning Agent · ช่วยวางแผนวัตถุดิบจาก SO

AI Agent

Agent ดึงข้อมูลจาก ERP / BOM / Stock ปัจจุบัน แล้วเสนอแผนการใช้วัตถุดิบ:

  • แตก BOM จาก Product Code → วัตถุดิบหลัก (เช่น Coil / แผ่นยาว)
  • เช็ค stock แต่ละลอท เพื่อแนะนำลอทที่ควรใช้ก่อน (FEFO / FIFO)
  • ประเมินว่า “ควรตัดเผื่อ” เท่าไหร่ เพื่อลดเศษวัสดุเหลือ

ตัวอย่าง Output (ให้ Planner / WMS ใช้ตัดสินใจ): 

{
  "so_no": "SO-2025-000121",
  "material_plan": [
    {
      "raw_material": "COIL-035-WH",
      "recommend_qty_kg": 1450,
      "from_location": ["RM-RACK-01", "RM-RACK-02"],
      "note": "ใช้ลอทใกล้หมดอายุก่อน · ลดเศษปลาย"
    }
  ]
}

5) Loading & Handover Agent · ช่วยหน้างานลานโหลด

AI Agent

Agent คอยดึงสถานะจาก WMS/ERP มาสรุปให้ง่ายสำหรับหัวหน้าลานโหลด และลูกค้าที่มารับของ.

  • เช็คว่ารายการใน Pick List ครบหรือยัง / ขาดอะไร
  • ทำ Check-list ให้ทีมโหลดติ๊กผ่านมือถือ / แท็บเล็ต
  • สร้างข้อความสรุปส่งให้ลูกค้าใน LINE OA ว่า “ของพร้อมรับแล้ว / เหลืออีกกี่รายการ”

ตัวอย่างข้อความที่ Agent ส่งไปยังลูกค้า:

✔️ ออเดอร์ SO-2025-000121 เตรียมสินค้าเรียบร้อยแล้ว
- แผ่นเมทัลชีท 0.35x6m สีขาว 120 แผ่น
- ครอบสันโค้งสีขาว 40 เส้น
สามารถเข้ามาตรวจรับและโหลดสินค้าได้ที่ลาน A ช่อง 3 ระหว่างเวลา 13.00–16.00 น.

6) Exception & Monitoring · เวลามีเคสผิดปกติระหว่างเตรียมของ

ถ้า stock ไม่พอ / วัตถุดิบไม่ตรงลอท / มีของเสียระหว่างผลิต Agent จะ:

  • แจ้งเตือนทีม Planner / Sales ผ่าน LINE หรือ Email
  • เสนอ Option แก้ไข เช่น “ลดจำนวนส่งรอบนี้” หรือ “เปลี่ยนสี/สเปกใกล้เคียง”
  • เก็บ Log ไว้ใน Exception Library เพื่อใช้เรียนรู้ในเคสถัดไป

Data Journey · 1 สินค้าตัวเดียวกัน ตั้งแต่ SO → แผนวัตถุดิบ → ตัด Stock → สรุปโหลดของ
