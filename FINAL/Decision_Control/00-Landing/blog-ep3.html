Beyond RAG: เจาะลึกสถาปัตยกรรม AI
ที่ "ห้ามคิดเลข" และ "โกงไม่ได้"

ทำไม Generative AI ถึงไม่พอสำหรับงาน Enterprise? เปิดพิมพ์เขียวระบบ "Neuro-Symbolic" ที่ผสาน LLM เข้ากับ Python Logic เพื่อความแม่นยำ 100%

10 min read
Technical Deep Dive

ในฐานะคนทำ Tech เราทุกคนรู้ความลับข้อหนึ่งของ LLM (Large Language Models) ที่ฝ่ายการตลาดไม่ค่อยพูดถึง:

"AI เก่งภาษาเหมือนกวี... แต่ตกเลขเหมือนเด็กอนุบาล"

ถ้าคุณให้ ChatGPT คำนวณภาษีซ้อนหรือตรวจสอบเงื่อนไขสัญญาที่ซับซ้อน มันอาจจะตอบคุณด้วยความมั่นใจ... แต่ผิด (Hallucination). สำหรับงานจัดซื้อระดับ Enterprise ความผิดพลาดแม้แต่ 0.1% คือความเสียหายหลักล้าน

ที่ TH8.AI เราจึงไม่ใช้ AI ธรรมดา แต่เราสร้างสถาปัตยกรรมแบบ "Neuro-Symbolic AI" บทความนี้จะพาคุณไปดูไส้ในของระบบ (Under the hood) ว่ามันทำงานอย่างไร

1 Neuro-Symbolic Architecture

เราออกแบบระบบโดยแยกการทำงานออกเป็น 2 ส่วนชัดเจน เหมือนสมองมนุษย์ซีกซ้ายและขวา:

Neural Layer (LLM)

  • หน้าที่: Orchestrator (ผู้สั่งการ)
  • ทำอะไร: อ่าน PDF, เข้าใจเจตนา (Intent), สรุปความ
  • ข้อห้าม: ห้ามคำนวณเงิน, ห้ามตัดสินใจอนุมัติ

Symbolic Layer (Code)

  • หน้าที่: Executor (ผู้ปฏิบัติงาน)
  • ทำอะไร: คำนวณส่วนต่าง (Delta), ตรวจสอบ Policy, บันทึก Log
  • จุดเด่น: Deterministic (1+1 ต้องเท่ากับ 2 เสมอ)

2 The "Kill Switch" for Hallucination

เราแก้ปัญหา AI มั่วตัวเลขด้วยเทคนิค "Native Tool Calling" แทนที่จะให้ LLM สร้างคำตอบเอง เราบังคับให้มันเรียกใช้ Python Function ที่เราเขียนไว้

ดูตัวอย่าง Flow การทำงานจริงเมื่อมีการตรวจสอบราคา:

policy_engine/core.py 
# 1. LLM detects intent but DOES NOT calculate.
# It returns a structured request to call a tool instead.

class PriceCheckTool(BaseTool):
    name = "calculate_price_delta"
    description = "Use this whenever user asks about price variance."

    def run(self, po_price: float, contract_price: float):
        # Pure Python Logic: 100% Deterministic
        delta = po_price - contract_price
        percent = (delta / contract_price) * 100
        
        if percent > 5.0:
            return {
                "status": "VIOLATION",
                "message": f"Price exceeds contract by {percent}%",
                "action": "BLOCK_APPROVAL"
            }
        return {"status": "PASS"}

*Code นี้ทำงานบน Server ที่ปลอดภัย ไม่ได้รันบน LLM Provider ดังนั้น Logic ธุรกิจของคุณจึงเป็นความลับและแม่นยำเสมอ
