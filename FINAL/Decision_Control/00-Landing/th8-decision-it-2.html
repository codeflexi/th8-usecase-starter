01 Context Engineering

We don't just "send prompts." We engineer a Dynamic Context Window. Before the LLM generates a single token, our middleware assembles a precise state snapshot containing only relevant facts to minimize hallucination.

System Prompt Injection

Role definition, tone guidelines, and strictly forbidden actions (e.g., "Do not approve if over budget").

RAG Fusion

Injecting retrieved contract clauses (Vector DB) + specific PO data (Structured DB) into the prompt context.

context_builder.py 
def build_agent_context(session_id, query):
    # 1. Retrieve Short-term Memory
    history = redis.get_conversation(session_id)
    
    # 2. Retrieve Long-term Knowledge (RAG)
    docs = vector_db.search(query, top_k=3)
    
    # 3. Construct System Message (The Guardrail)
    system_prompt = f"""
    You are TH8 Procurement Agent.
    Current Time: {datetime.now()}
    Active Policy: {policy_version_1_0}
    Retrieved Evidence: {docs}
    
    CRITICAL RULES:
    - Do NOT perform math yourself. Use 'calculate_delta' tool.
    - Do NOT approve deviations without 'check_policy' tool.
    """
    
    return [system_prompt, *history, query]

02 Agent Memory Architecture

TH8 Agents possess a dual-memory system mimicking human cognition. This ensures the agent remembers "what you just said" while knowing "what the company policy is."

  • Episodic Memory: (Redis) Keeps track of the multi-turn conversation state. e.g., "User wants to approve PO #002".
  • Semantic Memory: (Qdrant/Milvus) Stores static knowledge like Contracts, Policy PDFs, and Vendor History encoded as vectors.
  • Entity Memory: (Structured DB) Specific facts extracted from the current transaction (PO Number, Amount, Vendor Name).
No Black Box Math

Native Tool Calling & Rule Engine

We forbid the LLM from doing arithmetic or making final policy decisions. Instead, the LLM acts as a "Router" that calls deterministic Python functions (Tools) to execute the logic.

Hybrid Execution Flow

1. LLM Intent

"User asks if PO-002 price is correct." -> LLM decides to call calculate_delta.

2. Tool Execution (Python)

System runs: 12.80 - 12.50 = 0.30 (+2.4%).
The math is 100% accurate.

3. Response Synthesis

Tool result is fed back to LLM.
LLM responds: "The price is 2.4% higher than contract."

Why Deterministic?

Financial governance cannot rely on probability. We use a Symbolic Layer for all calculations and threshold checks.

Benefit

"Zero Hallucination"

The LLM explains the result, but it does not *compute* the result.

policy_engine/rules.py 
class PriceDeviationRule(BaseRule):
    """
    If PO Price > Contract Price by > 1%, trigger PENDING.
    If PO Price > Contract Price by > 5%, trigger HIGH_RISK.
    """
    def execute(self, po, contract):
        delta_pct = (po.unit_price - contract.unit_price) / contract.unit_price * 100
        
        if delta_pct > 5.0:
            return RuleResult(
                status="HIGH_RISK", 
                message=f"Exceeds threshold by {delta_pct}%"
            )
        elif delta_pct > 1.0:
            return RuleResult(status="PENDING")
            
        return RuleResult(status="PASS")
