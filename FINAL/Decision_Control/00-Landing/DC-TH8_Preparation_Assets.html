1. Data Privacy & AI Model Isolation

TH8.AI ยึดถือหลักการ "Your Data is Yours" เป็นหัวใจสำคัญในการออกแบบระบบ สถาปัตยกรรมของเราการันตีความปลอดภัยของข้อมูลลูกค้า ดังนี้:

  • Zero-Training Policy: ข้อมูล Transaction และเอกสารสัญญาของลูกค้า จะไม่ถูกนำไปใช้เทรน (Train) หรือปรับปรุงโมเดล AI สาธารณะ (Public LLM) เด็ดขาด
  • Single-Tenant Logical Isolation: ข้อมูลของลูกค้าแต่ละรายจะถูกแยกออกจากกัน (Siloed) ทั้งในระดับ Database และ Vector Store ไม่มีการปะปนข้าม Tenant
  • Stateless Processing: ข้อมูลที่ส่งมาประมวลผลผ่าน API จะถูกเก็บรักษาเท่าที่จำเป็นตามระยะเวลา Retention Policy ที่ลูกค้ากำหนด และสามารถตั้งค่าให้ลบอัตโนมัติได้ทันทีหลังประมวลผลเสร็จ

2. Encryption & Infrastructure

ระบบของเราทำงานบน Cloud Infrastructure มาตรฐานสากล (AWS/Azure) โดยมีมาตรการเข้ารหัสสูงสุด:

Data at Rest AES-256 Encryption สำหรับข้อมูลที่ถูกจัดเก็บใน Database และ Object Storage ทั้งหมด
Data in Transit TLS 1.2/1.3 สำหรับทุกการเชื่อมต่อผ่าน API และหน้าเว็บ (HTTPS enforced)

3. Access Control & Compliance

  • RBAC (Role-Based Access Control): ระบบมีการแบ่งสิทธิ์ผู้ใช้งานชัดเจน (Admin, Manager, Viewer) เพื่อจำกัดการเข้าถึงข้อมูลละเอียดอ่อน
  • Audit Logging: ทุกกิจกรรมในระบบ (Login, View Data, Change Rule) จะถูกบันทึกเป็น Immutable Logs ที่ไม่สามารถแก้ไขได้ เพื่อรองรับการตรวจสอบตามมาตรฐาน PDPA และ GDPR

Sales Tool

Objective: อย่าเพิ่งขายของ! ให้ใช้หน้านี้ "ถาม" เพื่อเก็บข้อมูลให้ได้มากที่สุด แล้วค่อยเอาข้อมูลนี้ไปตั้งค่า Demo ให้ตรงใจลูกค้า

1 Current Pain & Process

  • ปัจจุบันกระบวนการอนุมัติใช้เวลานานแค่ไหน? (SLA ปัจจุบันคือกี่วัน?)
  • เคสที่หลุดบ่อยที่สุดคืออะไร? (ซื้อแพง? เอกสารไม่ครบ? หรือดองงาน?)
  • มีระบบ ERP อะไรอยู่แล้ว? (SAP / Oracle / Microsoft / หรือ Excel ล้วน?)

2 Volume & Value (เพื่อทำ ROI)

  • เฉลี่ยมี PO/PR เดือนละกี่ใบ? (Transaction Volume)
  • ทีมจัดซื้อมีกี่คน? (Headcount)
  • ปีที่แล้วมีเคส "Maverick Spending" (ซื้อนอกระบบ/ซื้อด่วน) เยอะไหม? มูลค่าประมาณเท่าไหร่?

3 Decision Making Criteria

  • นอกจากราคากับฟีเจอร์แล้ว ซีเรียสเรื่องอะไรที่สุด? (Security? ความง่าย? หรือ Support?)
  • ปกติขั้นตอนการจัดซื้อซอฟต์แวร์ของบริษัทต้องผ่านใครบ้าง? (IT, Procurement, Legal?)
