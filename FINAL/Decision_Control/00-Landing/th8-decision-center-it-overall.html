Deterministic Governance Engine

เราเข้าใจความกังวลของ IT: "AI มั่วข้อมูล (Hallucination)" และ "ตรวจสอบไม่ได้ (Black Box)"

TH8 จึงถูกออกแบบด้วยสถาปัตยกรรม Hybrid: ใช้ LLM เพื่อความเข้าใจภาษา แต่ใช้ Hard-coded Logic ในการคำนวณและตัดสินใจ เพื่อให้ Audit ได้ 100%

core_logic_enforcement.py 
class ProcurementGuardrail(BaseModel):
    """Deterministic Policy Layer (No LLM Guessing)"""
    
    def evaluate(self, po, contract):
        # 1. RAG Evidence Retrieval
        evidence = vector_db.search(po.sku, contract.id)
        
        # 2. Hard Rule Check (Math is executed by CPU, not AI)
        violations = []
        delta = po.price - contract.price
        if delta > 0:
             if (delta / contract.price) > 0.05:
                 violations.append("HIGH_RISK_PRICE")
        
        # 3. Decision
        if violations:
            return Decision("PENDING", reason=violations)
        return Decision("AUTO_PASS")

A Dual-Memory Architecture

ระบบจำลองความจำมนุษย์เพื่อความแม่นยำสูงสุด:

  • Episodic Memory (Redis) จำ Context การสนทนาปัจจุบัน (Session State) ทำให้คุยต่อเนื่องได้ไม่สะดุด
  • Semantic Memory (Vector DB) จำ "ความรู้องค์กร" (Contracts, Policies) แบบถาวร ดึงมาใช้เมื่อเกี่ยวข้องเท่านั้น (RAG)

B Native Tool Calling (No Black Box)

Data Privacy

  • On-Premise Ready: รองรับการติดตั้งใน VPC หรือ Private Cloud ขององค์กร
  • PII Masking: ระบบสามารถ Mask ข้อมูลส่วนบุคคลก่อนส่งไปประมวลผลที่ LLM

Integration Patterns

ออกแบบให้ทำงานร่วมกับ ERP เดิม (SAP, Oracle) โดยไม่กระทบ Core System รองรับ 2 รูปแบบ:

API

Synchronous (Pre-Check)

เรียก POST /v1/policy/check ก่อนกด Save PO เพื่อรับสถานะ Pass / Pending ทันที

HOOK

Asynchronous (Webhooks)

สำหรับงานหนัก (เช่น แนบ PDF สัญญาใหม่) ส่งผ่าน API แล้วระบบจะ Callback กลับไปเมื่อประมวลผลเสร็จสิ้น
