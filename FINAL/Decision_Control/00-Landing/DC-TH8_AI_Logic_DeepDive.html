Logic & Prompt Specification

Deep Dive into Rule-Based vs LLM Reasoning

1. S&OP Harmonizer (Sales vs Ops)

Hard Logic (Rule-Based)

คำนวณความเป็นไปได้ทางกายภาพ (Physical Constraints)

Capacity_Gap = Forecast_Qty - (Run_Rate_Per_Day * Available_Days)
Material_Status = IF(Inventory < BOM_Req, "SHORTAGE", "OK")

Output: ตัวเลขขาด/เกิน ที่แม่นยำ 100%

Soft Logic (LLM Reasoning)

เสนอทางเลือกเชิงกลยุทธ์ (Trade-off Analysis)

SYSTEM: You are a Supply Chain Strategist. CONTEXT: - Shortage: 5,000 units of PET Bottles. - Supplier Lead Time: 30 days. - Campaign Starts: 15 days. - Penalty Cost for Rush Order: +20% price. - Opportunity Loss if delay: 1M THB. TASK: Propose 3 scenarios to solve this conflict. Calculate financial impact for each. Recommend the best option for maximize EBITDA.

Output: แผน A (จ่ายแพงซื้อด่วน), แผน B (เลื่อนโปรโมชั่น), แผน C (เปลี่ยน SKU)

2. Capex Scout (Investment)

Hard Logic (Rule-Based)

คำนวณตัวเลขทางการเงิน (Financial Ratios)

NPV = Σ [ CashFlow / (1 + r)^t ] - Initial_Investment
Payback_Period = Initial_Investment / Annual_Cash_Inflow

Soft Logic (LLM Reasoning)

ตรวจสอบสมมติฐานความจริง (Reality Check)

SYSTEM: You are a Senior Investment Auditor. INPUT CLAIM: "New Robot Arm will reduce 20 headcount." HISTORICAL DATA: Similar project 'Robot Gen 1' in 2022 reduced only 12 headcount due to frequent breakdowns. TASK: Validate the claim. If overly optimistic (>10% variance from history), flag it as 'High Risk' and recalculate ROI using conservative numbers (12 headcount).

3. Launch Control (Marketing Compliance)

Hard Logic (Rule-Based)

กรองคำต้องห้าม (Blacklist Keywords)

Violation = IF(Text contains ["ขาวถาวร", "หายขาด", "รับประกันผล"], TRUE, FALSE)

Soft Logic (LLM Reasoning)

เข้าใจบริบทกฎหมายและแนะนำคำใหม่ (Legal Re-writing)

SYSTEM: You are an expert in Thai FDA & Consumer Protection Law. INPUT AD: "ครีมนี้ทาแล้วขาวถาวร เห็นผลใน 3 วัน" LEGAL CONTEXT: Cosmetic Act forbids claiming permanent physiological changes. TASK: 1. Identify legal violations. 2. Rewrite the copy to be compliant but still attractive for marketing. 3. Example rewrite: "ช่วยให้ผิวดูกระจ่างใสขึ้นเมื่อใช้เป็นประจำ"

4. Shift Commander (Workforce)

Hard Logic (Rule-Based)

กฎหมายแรงงาน (Labor Law Constraints)

Eligibility = IF(Work_Hours_Week < 48 AND Last_Shift_End > 10_Hours_Ago, "ELIGIBLE", "BLOCKED")

Soft Logic (LLM Reasoning)

ทำนายพฤติกรรมคน (Behavioral Prediction)

SYSTEM: You are a Shift Manager Assistant. CONTEXT: Need 5 people for urgent Saturday shift. CANDIDATE POOL: - Somchai: Accepts Sat shifts 90% of the time. - Somsri: Has small children, rejects 80% of weekend shifts. TASK: Rank candidates by 'Likelihood to Accept' to minimize time spent calling. Prioritize high performers who need extra income.

5. Voice-to-Feature (Product Loop)

Hard Logic (Rule-Based)

คำนวณมูลค่าความเสียหาย (Revenue Impact)

Impact_Value = Count(Unique_Complainers) * Avg_Customer_LTV

Soft Logic (LLM Reasoning)

สรุปใจความสำคัญ (Semantic Clustering)

SYSTEM: You are a Product Owner. INPUT: 500 chat logs from yesterday. TASK: 1. Cluster complaints into topics (e.g., UI Bug, Payment Fail, Feature Req). 2. For the biggest cluster, write a 'User Story' format: "As a [User], I want to [Action], so that [Benefit]." 3. Identify emotional sentiment (Frustrated vs Disappointed).
