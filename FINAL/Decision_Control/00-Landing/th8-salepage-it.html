class ProcurementGuardrail(BaseModel):
    """Deterministic Policy Enforcement Layer"""
    
    async def evaluate_transaction(self, po: PurchaseOrder, contract: Contract) -> Decision:
        # 1. Vector Search for Evidence (RAG)
        evidence = await self.vector_store.similarity_search(
            query=po.item_description, 
            filter={"contract_id": contract.id}
        )
        
        # 2. Hard Rule Check (No LLM Hallucination here)
        violations = []
        if po.unit_price > contract.unit_price * (1 + self.threshold):
            violations.append(
                RuleViolation(code="R-01", severity="HIGH", delta=po.unit_price - contract.unit_price)
            )

        # 3. Decision Logic
        if not violations:
            return Decision(status="AUTO_PASS")
        return Decision(status="PENDING", reason=violations, evidence=evidence)
Core Capabilities

The Technology Engine

Our "Engine" is not a single model, but an orchestrated pipeline of specialized components designed to handle complexity, ambiguity, and strict compliance simultaneously.

1. Cognitive Extraction (OCR + LLM)

Handles unstructured data ingestion. We use optical character recognition (OCR) combined with LayoutLM models to understand complex tables in PDF contracts.

  • Table Structure Recognition
  • Multi-lingual Support (TH/EN)
  • Evidence Snippet Generation

2. Context-Aware RAG

Retrieval-Augmented Generation that grounds every answer in your specific documents. We use hybrid search (Dense Vector + Sparse Keyword) for maximum accuracy.

  • Qdrant / Milvus Vector Store
  • Similarity Re-ranking
  • Provenance Tracking (Source linking)
CORE LOGIC

3. Policy Engine (Symbolic)

Unlike pure LLMs, our Policy Engine is deterministic. It executes hard business rules (e.g., "Price > Contract + 5%") that cannot be bypassed by "prompt injection".

  • Code-based Rule Execution
  • Zero Hallucination on Math
  • Configurable Risk Thresholds

System Architecture

A decoupled, event-driven architecture designed for high throughput and fault tolerance. Heavy lifting (PDF Parsing) is offloaded to worker nodes, while the API Gateway remains responsive.

Scalability Strategy

Horizontal Scaling: The API Layer and Worker Nodes (Celery) scale independently based on load (HTTP reqs vs. PDF ingestion queue).

Resilience

Circuit Breakers: Implemented on LLM API calls and Vector DB queries to prevent cascading failures.

  • On-Premise Ready: Can be deployed on your private cloud (AWS VPC, Azure VNet) or on-premise Kubernetes.
  • Encryption: AES-256 for data at rest, TLS 1.3 for data in transit.

Integration Patterns

TH8.AI is designed to coexist with your existing ERP. We support multiple integration patterns depending on your latency requirements.

API

Synchronous (Pre-Check)

Call our POST /v1/policy/check endpoint before PO creation. Returns Pass / Pending status instantly.

HOOK
