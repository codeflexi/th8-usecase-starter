TH8.AI · Enterprise Decision Control Layer

Technical Implementation Blueprint

เอกสารนี้อธิบายว่า TH8 ต้อง implement อะไรบ้าง เพื่อให้ demo เห็นภาพ end-to-end จริง ทั้งเชิงเทคนิคและเชิงธุรกิจ โดยไม่แทนที่ SAP / Workflow เดิม

Part A — Technical Implementation Checklist (Backoffice)

1) Integration & Event Ingestion

  • • Standard Event API (/events)
  • • Event schema versioned
  • • Idempotency (กัน event ซ้ำ)
  • • Source system tagging (SAP / WF / MANUAL)

Business goal: TH8 “รู้ทุกอย่างที่เกิดขึ้น” โดยไม่ execute workflow

2) Case Correlation Engine

  • • Map PO / Invoice / Workflow → 1 Case
  • • Business key correlation
  • • Unified case lifecycle

Business goal: ผู้บริหารคิดเป็น “เรื่องเดียว”

3) Decision State Machine

  • • OPEN / PENDING / ESCALATED
  • • OVERRIDDEN / RESOLVED
  • • Decision ≠ Transaction

Business goal: แยก “ทำอะไร” ออกจาก “ตัดสินใจอย่างไร”

4) Policy Evaluation Engine

  • • Authority threshold
  • • Contract clause rules
  • • SLA & override logic
  • • Policy version binding

Business goal: AI ทำตาม policy ไม่ใช่คิดเอง

5) Evidence / RAG as Evidence

  • • Allowed document sources
  • • Clause / page citation
  • • Evidence linked to decision

Business goal: ทุก decision มีหลักฐาน

6) Audit Timeline Engine

  • • Cross-system timeline
  • • Source-tagged events
  • • Immutable audit log

Business goal: audit จบใน TH8

Part B — ERD (Decision & Audit Model)

Case
 ├─< CaseEvent
 ├─< Decision
 ├─< Evidence >─ Document
 └─< AuditLog

Policy
 └─ referenced by → Decision / CaseEvent

Case

หน่วยธุรกิจหลักที่ผู้บริหารมองเป็น 1 เรื่อง

Decision

การตัดสินใจเชิงบริหาร (approve / override / reject)

CaseEvent

Event จากทุกระบบที่ถูก normalize

AuditLog

Log สำหรับ audit แบบ immutable

Executive & IT Takeaway

TH8 ไม่ได้มาแทน SAP หรือ Workflow

แต่ทำให้ทุกการอนุมัติ “มีข้อมูลครบก่อนกด” และทุกการตัดสินใจ “ตรวจสอบย้อนหลังได้” แม้ action จะเกิดในระบบเดิมก็ตาม

ใช้เอกสารนี้เป็น blueprint สำหรับ demo, internal alignment และ enterprise IT discussion
